Project Overview

A production-ready MLOps pipeline for healthcare prediction

About This Project

This project presents a Strategic Proof-of-Concept (PoC) focused on building an MLOps pipeline using the MIMIC-III Clinical Database Demo (v1.4). While the demo dataset limits the statistical power of the results, it provides a complex, realistic environment to demonstrate an enterprise-ready MLOps architecture capable of handling real-world healthcare data challenges. The primary goal is showcasing methodological rigor, software engineering best practices, and an end-to-end workflow designed for reliability, reproducibility, and scalability.

Focus on Process & Scalability: The pipeline architecture, code structure, and MLOps components (testing, monitoring, deployment) are the key deliverables. Designed for the full MIMIC dataset scale and production EHR systems, the performance metrics shown are illustrative of the insights achievable via this robust process.

Scalability Considerations

Transitioning from this PoC to the full MIMIC dataset or a production EHR system involves addressing key scalability challenges:

Data Engineering at Scale
  • ETL: Utilize distributed processing frameworks like Apache Spark or Dask for handling terabytes of raw data efficiently.
  • Feature Stores: Implement feature stores such as Feast or Tecton for managing versioned features, ensuring consistency between training and serving, and enabling feature reuse.
  • Data Validation: Employ robust data validation tools (e.g., Great Expectations) within the pipeline to handle data quality issues at scale.
Training & Infrastructure
  • Distributed Training: Leverage frameworks like Horovod or PyTorch DDP for training complex models (like LSTMs/Transformers) on large datasets across multiple GPUs/nodes.
  • Hyperparameter Tuning: Employ efficient, parallelized tuning libraries like Ray Tune or Optuna.
  • Infrastructure: Utilize managed cloud ML platforms (AWS SageMaker, GCP Vertex AI, Azure ML) or Kubernetes for scalable resource provisioning and orchestration (including GPU management).
Monitoring at Scale

Drift detection and performance monitoring must handle high throughput. This involves:

  • Specialized monitoring tools (e.g., Arize, Fiddler, WhyLabs) or robust custom statistical process control (SPC) implementations.
  • Efficient logging and aggregation strategies for handling millions of daily predictions.
  • Automated alerting based on predefined drift thresholds and performance degradation.

Key Features & Innovations

Advanced techniques that set this project apart

Advanced Temporal Modeling

Leveraging the sequential nature of EHR data using a Time-Aware LSTM with Attention (see notebooks/time_aware_lstm.py). This approach captures irregular time intervals and identifies critical time points.

class TimeAwareLSTM(nn.Module):
    # ... (Initialization) ...
    def forward(self, x, time_intervals):
        # Incorporate learned time embeddings
        # based on irregular measurement intervals
        time_encoded_x = self.time_encoder(x, time_intervals)

        lstm_out, _ = self.lstm(time_encoded_x)

        # Attention mechanism identifies key time steps
        context, attn_weights = self.attention(lstm_out)

        output = self.classifier(context)
        return torch.sigmoid(output), attn_weights

This enhanced model (advanced_temporal_model_enhanced.ipynb) demonstrates improved performance over static baselines (e.g., LightGBM) by explicitly modeling time dynamics, a critical factor often ignored.

Causal Inference Exploration

Exploring techniques to estimate the causal effect of interventions or features on readmission, moving beyond simple correlations found in observational data.

Technique Example: Utilized Doubly Robust Estimation (via libraries like EconML) to estimate the Average Treatment Effect (ATE) of a specific intervention (e.g., early mobility protocol) while controlling for measured confounders present in the observational MIMIC data.

Preliminary Finding (Highly Illustrative): Initial analysis on this demo subset suggests the intervention has an estimated ATE of -0.03 (a 3% reduction) on 30-day readmission probability. This is purely illustrative due to data limitations.

Next Steps for Robustness: Definitive causal claims require significant further work, including validation on larger datasets, rigorous sensitivity analysis (e.g., testing robustness to unmeasured confounders), and potentially exploring alternative causal identification strategies.

Going beyond basic implementations

Model Monitoring & Observability

  • Data Drift Detection: Continuous monitoring using metrics like Population Stability Index (PSI) and statistical tests (e.g., KL Divergence) for key features.
  • Concept Drift Detection: Tracking outcome metrics (e.g., AUC, F1) on incoming data streams compared to validation performance to detect performance degradation.
  • Operational Metrics: Monitoring API latency, error rates, and throughput.
  • Tooling: Leveraging logging frameworks, Prometheus/Grafana for dashboards, and potentially integrating with MLflow for drift tracking.

Project Visualizations

Key insights from our analysis and modeling

Imbalance Handling Techniques Comparison

One of the key challenges in healthcare prediction is dealing with class imbalance. In our dataset, only 12.1% of admissions resulted in readmission, creating a 7.2:1 imbalance ratio. We implemented and compared several techniques to address this challenge:

Key Findings:

  • Baseline Failure: Without imbalance handling, the model failed to predict any positive cases
  • SMOTE Superiority: SMOTE provided the best overall performance with the highest F1 score (0.373), indicating a better balance between precision and recall compared to other methods on this dataset.
  • Precision-Recall Trade-off: Class weights and random oversampling achieved high recall (>0.81) but at the cost of precision (<0.20)
  • Synthetic vs. Duplicated Samples: SMOTE's synthetic samples led to better generalization than random oversampling's duplicated samples

Interactive Precision-Recall Curves:

The PR curves illustrate the trade-off between precision and recall across different classification thresholds. SMOTE (green) maintains better precision across a wider range of recall values compared to other techniques.

This visualization is particularly important in healthcare contexts where the cost of false negatives (missed readmissions) must be balanced against the cost of false positives (unnecessary interventions).

Try hovering over the curves to see exact values at each threshold.

Ethical AI Considerations

Addressing bias and fairness in healthcare AI

Applied Fairness Evaluation

We go beyond simply stating ethical principles by actively measuring and addressing potential biases using established fairness metrics:

Fairness Metrics & Results

Evaluation: We evaluated fairness using metrics like Equalized Odds Difference across the 'gender' attribute to quantify disparities in True Positive Rates between subgroups.

Initial Finding: The baseline LightGBM model showed a difference of 0.08 in True Positive Rates between male and female subgroups.

Mitigation/Monitoring: While specific mitigation techniques (like re-weighing or adversarial debiasing) are planned for the full-scale implementation, this metric is actively monitored in our evaluation pipeline. Current focus is on ensuring transparency around existing disparities.

Explainability in Action (XAI)

SHAP Integration: We integrated SHAP (SHapley Additive exPlanations) to provide both local (per-prediction) and global feature importance insights, enhancing model transparency.

Value Proposition: SHAP values allow clinicians to understand the key drivers (e.g., specific lab results, length of stay) contributing to an individual patient's risk score, fostering trust and enabling more informed decision-making.

Example: SHAP summary plot showing global feature importance (Click to enlarge)

Fairness-Performance Trade-offs: Our analysis revealed an inherent tension between demographic parity and overall model performance. We chose to prioritize equal opportunity (similar recall across groups) over demographic parity, as missing high-risk patients has more severe clinical consequences than false positives. This decision was made in consultation with clinical stakeholders and is continuously reassessed.

Future Directions & Enhancements

Leveraging the 2025 AI Landscape

This PoC establishes a strong foundation. Future iterations could incorporate cutting-edge AI advancements:

  • LLMs for Feature Extraction: Utilize Large Language Models (e.g., fine-tuned BioBERT or domain-specific models) to extract structured features from unstructured clinical notes, potentially uncovering novel predictive signals.
  • RAG for Explainability: Implement Retrieval-Augmented Generation (RAG) systems using vector databases (e.g., Pinecone, Weaviate) to retrieve relevant medical literature or similar patient cases, providing clinicians with evidence-based context alongside model predictions.
  • Enhanced Temporal Models: Explore Transformer-based architectures for capturing longer-range dependencies in patient histories.
  • Federated Learning: Investigate federated learning approaches for training models across multiple institutions without sharing raw patient data, addressing privacy concerns.
  • Synthetic Data Augmentation: Refine the use of synthetic data generation (notebooks/synthetic_data_generator.py) specifically for augmenting underrepresented patient subgroups or rare event scenarios to improve model robustness and fairness.
